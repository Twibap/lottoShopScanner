# 판매점 좌표 데이터 감사

## 목적

주변 판매점 검색 API를 구현하기 전에 판매점 좌표가 공간 검색에 사용 가능한지 반복적으로 검사한다.

## 검사 항목

- 위도 또는 경도 누락
- 숫자가 아니거나 유한하지 않은 좌표
- `(0, 0)` 좌표
- 대한민국을 넉넉하게 포함하는 범위(위도 32~39, 경도 124~132) 밖의 좌표
- 여러 판매점이 완전히 같은 좌표를 사용하는 경우
- 온라인 판매 사이트처럼 실제 방문 장소가 아닐 가능성이 있는 레코드

동일 좌표는 같은 건물에 여러 판매점이 있거나 과거·현재 판매점 ID가 함께 남은 경우일 수 있으므로 자동 오류로 판정하지 않는다. 주소 지오코딩 결과와 원천 데이터를 대조한 뒤 병합 또는 유지한다.

## 실행

```powershell
python .\apps\data-collector\src\audit_shop_coordinates.py `
  --input .\apps\data-collector\output\shop_rankings.jsonl `
  --output .\apps\data-collector\output\coordinate_audit.json
```

누락·비숫자·0·국내 범위 밖 좌표가 있으면 종료 코드 `1`, 입력을 읽지 못하면 `2`를 반환한다. 동일 좌표 및 비실물 후보는 검토 대상으로 보고서에 포함하지만 감사 실행을 실패시키지 않는다.

국내 범위 밖 좌표 4개는 주변 검색 대상에서 제외하고 주소 재지오코딩 대상으로 보낸다. 비실물 후보인 동행복권 인터넷 판매 사이트도 주변 검색 대상에서 제외한다. 동일 좌표 69개 그룹은 네이버 Geocoding 키가 준비된 후 주소 기반 좌표와 비교해 정확도를 추가 검증한다.

## API 구현 차단 조건

- 누락·비숫자·0·국내 범위 밖 좌표는 공간 인덱스에 포함하지 않는다.
- 비실물 판매점 후보는 주변 판매점 조회 결과에서 제외한다.
- 좌표 품질 상태와 제외 사유를 DB에 저장해 원천 데이터를 덮어쓰지 않고 재처리할 수 있어야 한다.
